# Workflow: Feature

## 1. Spec-Gate
- Сформулировать цель фичи в 1-2 предложениях.
- Определить scope / out-of-scope.
- Зафиксировать acceptance criteria.

## 2. Execution-Gate
- Внести изменения минимальным рабочим шагом.
- Проверить линтеры и релевантные тесты.
- Обновить документацию/контракты при изменении поведения.

## 3. Review-Gate
- Провести независимое ревью.
- Получить решение `PASS/FAIL`.
- Для `FAIL` устранить блокирующие замечания.

## 4. Merge-Gate
- Убедиться, что acceptance criteria выполнены.
- Приложить артефакты проверок.
- Выполнить merge в `main` только при PASS.
