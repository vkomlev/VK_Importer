- Name: VK_Importer
- Stack: Python, Telegram export ingestion, VK publishing

## Назначение
Этот документ задает обязательные правила для AI-агентов в проекте.

## Роли агентов
- Tier L: Cursor agents.
- Tier M: Codex.
- Tier H: Claude.

## Project Overrides
Проектные отклонения и особенности фиксируются в docs/ai/PROJECT_OVERRIDES.md.
Если есть конфликт между общими правилами и overrides, приоритет у overrides.

## Непереговорные правила
1. Не выполнять разрушительные действия без явного запроса.
2. Все изменения должны быть проверяемыми (команды, тесты, логи, артефакты).
3. для интеграции в main/master (merge или прямой commit) обязателен `review-gate`.
4. Изменения схемы БД только через миграции проекта.
5. Секреты не писать в код, логи, коммиты, документацию.

## Обязательный цикл работы
1. spec-gate: формализация задачи и критериев готовности.
2. execution-gate: реализация и минимальные проверки.
3. `review-gate`: независимое решение PASS/FAIL для интеграции в main/master (merge или прямой commit).
4. `merge-gate`: интеграция в main/master (merge или прямой commit) только при PASS.

## Артефакты по задаче
- План и критерии (spec).
- Список изменений (файлы/дифф).
- Результаты проверок (smoke/test/lint).
- Решение review-gate (PASS/FAIL + замечания).

## Политика веток и репозиториев
- Основной режим: работа напрямую в `main`/`master`.
- Отдельные ветки создавать только для полностью автономных запусков агентов (длительные/массовые изменения без участия человека в цикле).
- Для обычной совместной работы (человек + агент) ветки не обязательны.
- При работе в `main`/`master` обязательно усилить контроль перед commit:
  - проверить релевантные тесты/smoke
  - пройти `review-gate` для интеграции в `main`/`master` (merge или прямой commit)
  - убедиться, что изменения ограничены scope задачи
