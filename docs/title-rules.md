# Правила именования заголовков

Используются две группы генераторов (DRY): **экзаменационный стиль** и **префикс + описание**.

## Группа 1: ЕГЭ, ОГЭ, Алгоритмы

**Принцип:** различение «тема» / «задание», короткие темы — первые два предложения, указание источника (Решу ЕГЭ/ОГЭ, КЕГЭ и т.д.).

- **ЕГЭ:** `EGEAutoTitleGenerator` — тема или задание с номером (N_NNNN), первые 2 предложения для коротких тем, ресурс в скобках.
- **ОГЭ:** `OGEAutoTitleGenerator` — то же для ОГЭ (Решу ОГЭ, Поляков и т.д.).
- **Алгоритмы:** `AlgorithmsAutoTitleGenerator` — «Задание N.N» или тема из первой строки, при отсутствии описания — из stem (например «2.4 a7edf3» → «Задание 2.4»).

Общая логика: выбор между разбором темы и разбором задания по паттернам в описании; для коротких формулировок — расширение до двух предложений.

## Группа 2: Python, Excel, Аналитика данных, Комлев

**Принцип:** префикс курса/канала + первая строка описания (или имя файла при пустом описании).

- **Python:** отдельный генератор (`PythonAutoTitleGenerator`) — тема/задание, мини-уроки, падежи.
- **Excel:** `CoursePrefixTitleGenerator("Курс Excel")` — «Курс Excel. \<первая строка или stem\>».
- **Аналитика данных:** `CoursePrefixTitleGenerator("Аналитика данных")` — «Аналитика данных. \<первая строка или stem\>».
- **Комлев:** `CoursePrefixTitleGenerator("Канал Виктора Комлева")` — «Канал Виктора Комлева. \<первая строка или stem\>».

Ограничение длины заголовка (обрезание по словам) общее для префикс-генераторов.

## Пересчёт заголовков

- По всем каналам (включая новые): `python main.py recalc-titles`
- По одному каналу: `python main.py recalc-titles --channel Excel`

После добавления маппингов папок или смены правил пересчёт обновляет только поле `title` в БД.
