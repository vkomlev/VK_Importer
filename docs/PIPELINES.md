# Использование в пайплайнах (OpenClaw)

Публикатор предназначен для запуска из скриптов и пайплайнов OpenClaw. Ниже — сценарии, переменные окружения и особенности поведения.

## Переменные окружения

| Переменная | Обязательна для | Описание |
|------------|------------------|----------|
| `VK_ACCESS_TOKEN` | Загрузка и обновление в VK | Токен доступа VK API (пользовательский, с правами video, groups) |
| `VK_GROUP_ID` | Загрузка в группу | ID группы VK (число) |

Файл `.env` в корне проекта подхватывается автоматически; переменные из окружения процесса имеют приоритет над `.env`.

## Типичные шаги пайплайна

### 1. Настройка маппинга папок (один раз или при появлении новых каналов)

```bash
python main.py folders set "d:/path/to/export_folder" ЕГЭ
python main.py folders list
```

Маппинг хранится в БД (`videos.db`), менять код не нужно.

### 2. Сканирование экспортов (добавление видео в БД)

Полное сканирование всех папок из маппинга:

```bash
python main.py scan -s mapped
```

Инкрементальное сканирование по датам (только видео с датой в диапазоне):

```bash
python main.py scan -s mapped --since 2026-02-01 --until 2026-02-19
```

Конкретная папка:

```bash
python main.py scan -s "d:/path/to/export"
```

### 3. Пересчёт заголовков (по правилам курса)

```bash
python main.py recalc-titles
# или только для канала
python main.py recalc-titles --channel Excel
```

### 4. Загрузка в VK

Требуются `VK_ACCESS_TOKEN` и `VK_GROUP_ID`.

Одно следующее видео:

```bash
python main.py upload-next --channel ЕГЭ --delay 5
```

Диапазон по ID:

```bash
python main.py upload-range 1 --count 10 --delay 5
```

Все не загруженные:

```bash
python main.py upload-all --channel ЕГЭ --delay 5
```

### 5. Обновление заголовков в VK (после пересчёта в БД)

Список ID по умолчанию: `logs/titles_recalc_affected_ids.txt` (через запятую). Обновляются только уже загруженные в VK записи.

```bash
python main.py update-vk-titles --delay 15
```

## Логи и коды выхода

- **Лог-файл:** `logs/publisher.log` (UTF-8). Все важные операции и ошибки пишутся и в лог, и в stderr.
- **Уровень логирования:** задаётся переменной окружения `LOG_LEVEL` (DEBUG, INFO, WARNING, ERROR). По умолчанию INFO.
- **Коды выхода:** при критических ошибках (нет VK_ACCESS_TOKEN, неверный формат даты, отсутствует файл списка ID) команды завершаются с кодом 1. В остальных случаях — 0.

## Идемпотентность

- **scan** — можно вызывать повторно: существующие записи обновляются (заголовок, описание), дубликаты по хешу не создаются.
- **recalc-titles** — перезаписывает только поле заголовка в БД.
- **folders set** — перезаписывает маппинг для указанного пути.
- **upload-*** и **update-vk-titles** — не идемпотентны: меняют состояние в VK и в БД.

## Пути по умолчанию

- База данных: `videos.db` в текущей рабочей директории (откуда запускается `main.py`).
- Файл со списком ID для update-vk-titles: `logs/titles_recalc_affected_ids.txt`.

Запуск из корня проекта гарантирует ожидаемые пути.
