# Project Overrides - VK_Importer

## Цель
Фиксировать особенности импортера и публикации видео в VK.

## Доменные ограничения
- Сохранять консистентность статусов загрузки в SQLite.
- Соблюдать лимиты и retry-политику VK API.
- Не нарушать контракт команд CLI (`scan/stats/upload-*`).

## Ограничения по БД/интеграциям
- База `videos.db` — источник состояния pipeline.
- Внешняя интеграция VK: изменения только в пределах явно заданного сценария.
- Массовые операции публикации/удаления только через управляемые команды с логированием.

## Обязательные проверки перед merge в main
- Прогнать `scan`/`stats` smoke для затронутого кейса.
- Проверить корректность фильтров (channel/source/date) при затронутых изменениях.
- Проверить логи и корректность статусов `uploaded/error`.
- Получить `review-gate: PASS`.

## Ограничения по безопасностным действиям
- Не выводить VK токены и чувствительные данные в логах.
- Не запускать удаление контента из VK без явного запроса.

## Исключения и особые кейсы
- Для аварийного восстановления допустимы точечные ручные операции в БД, если это явно зафиксировано в задаче.
