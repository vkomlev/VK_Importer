# Аудит: документация, пайплайны OpenClaw, логирование и CLI

## 1. Резюме

Публикатор готов к использованию в пайплайнах (скрипты, OpenClaw) при учёте ограничений: **нет явных кодов выхода при ошибках**, часть документации устарела. Ниже — результаты аудита и рекомендации.

## 3. Использование в пайплайнах (OpenClaw)

### 3.1 Подходящие команды для пайплайнов

| Команда | Идемпотентность | Требует VK API | Рекомендация |
|---------|------------------|----------------|--------------|
| `scan` | Да (повторный скан обновляет записи) | Нет | Подходит для шага «добавить видео в БД» |
| `scan -s mapped --since / --until` | Да | Нет | Подходит для инкрементального скана по датам |
| `folders list` | Да | Нет | Проверка маппинга |
| `folders set` | Да | Нет | Настройка маппинга без кода |
| `stats` | Да | Нет | Проверка состояния БД |
| `recalc-titles` | Да | Нет | Пересчёт заголовков в БД |
| `upload-next`, `upload-one`, `upload-range`, `upload-many`, `upload-all` | Нет (меняют БД и VK) | Да | Шаг публикации в VK |
| `update-vk-titles` | Нет (меняет VK) | Да | Синхронизация заголовков в VK |

### 3.2 Переменные окружения

| Переменная | Обязательна | Где используется |
|------------|-------------|------------------|
| `VK_ACCESS_TOKEN` | Для загрузки и обновления в VK | upload-*, update-vk-titles |
| `VK_GROUP_ID` | Для загрузки в группу | upload-* |
| `LOG_LEVEL` | Нет | Уровень логирования: DEBUG, INFO, WARNING, ERROR (по умолчанию INFO) |

Остальные параметры (пути к БД, папки экспорта) задаются аргументами CLI или берутся по умолчанию (`videos.db`, текущая директория).

### 3.3 Коды выхода

- **Реализовано:** при критических ошибках команды `scan` (неверный формат `--since`/`--until`) и `update-vk-titles` (нет токена, файл не найден) выполняется `sys.exit(1)`.
- **Остальные команды:** при ошибках пока только `return` (код 0). Для полной поддержки пайплайнов имеет смысл добавить exit(1) при отсутствии VK_ACCESS_TOKEN/VK_GROUP_ID в upload-*.

### 3.4 Логирование при использовании в пайплайнах

- Лог-файл: `logs/publisher.log` (кодировка UTF-8).
- В консоль идут и вывод `click.echo`, и сообщения `logging` (уровень INFO).
- Для отладки пайплайнов можно повысить уровень до DEBUG через переменную окружения (см. раздел 5).

## 4. Справочник CLI (полный)

### 4.1 Верхний уровень

- `python main.py --help` — список команд.
- Все подкоманды: `scan`, `stats`, `upload-one`, `upload-next`, `upload-range`, `upload-many`, `upload-all`, `recalc-titles`, `update-vk-titles`, `folders`.

### 4.2 Команды и опции

| Команда | Аргументы | Опции | Описание |
|---------|-----------|--------|----------|
| **scan** | — | `-s`/`--source` (all \| all_channels \| ege \| python \| oge \| mapped \| tg_parser \| путь), `--since` YYYY-MM-DD, `--until` YYYY-MM-DD | Сканирование экспортов, добавление/обновление видео в БД |
| **stats** | — | — | Статистика по БД |
| **upload-one** | `video_id` | `-d`/`--delay`, `-r`/`--max-retries` | Загрузить одно видео по ID |
| **upload-next** | — | `-c`/`--channel`, `-s`/`--source`, `-d`, `-r` | Следующее не загруженное видео |
| **upload-range** | `start_id` | `-n`/`--count`, `-c`, `-s`, `-d`, `-r` | Загрузка диапазона по ID |
| **upload-many** | — | `-n`/`--count`, `-c`, `-s`, `-d`, `-r` | Несколько не загруженных видео |
| **upload-all** | — | `-c`, `-s`, `-d`, `-r` | Все не загруженные видео |
| **recalc-titles** | — | `-c`/`--channel` | Пересчёт заголовков в БД по правилам курса |
| **update-vk-titles** | — | `--ids-file`, `-d`/`--delay` | Обновление заголовков в VK по списку ID |
| **folders list** | — | — | Список маппингов папка → курс |
| **folders set** | `folder_path`, `course_type` | — | Установить маппинг |
| **folders remove** | `folder_path` | — | Удалить маппинг |

Общие опции загрузки: `--delay` (по умолчанию 5 сек, для update-vk-titles 15 сек), `--max-retries` (3).

### 4.3 Пробелы в CLI

- Нет единой опции `--verbose`/`-v` и `--quiet` для всего приложения.
- Нет явной опции `--db` для пути к БД (всегда `videos.db` в текущей директории).
- Список курсов для `folders set` зашит в константу (COURSE_TYPES) — расширение только правкой кода или БД.

## 5. Логирование

### 5.1 Текущая настройка

- **Файл:** `logs/publisher.log`, кодировка UTF-8.
- **Консоль:** тот же уровень и формат.
- **Формат:** `%(asctime)s - %(name)s - %(levelname)s - %(message)s`.
- **Уровень:** INFO (жёстко в `main.py`).

### 5.2 Где используется logger

- **src/storage/database.py** — инициализация БД, маппинг, добавление/обновление записи, update_description.
- **src/storage/scanner.py** — обход экспортов, количество обработанных видео, пропуск по дате, предупреждения (путь не существует, формат не определён).
- **src/publisher/vk_publisher.py** — загрузка, повторы, ошибки API, обновление заголовка (edit_video_title).
- **src/parsers/** — парсинг файлов, предупреждения (файл не найден, ошибки парсинга).

### 5.3 Пробелы (частично закрыты)

- **Реализовано:** уровень логирования задаётся переменной окружения `LOG_LEVEL` (DEBUG, INFO, WARNING, ERROR).
- Нет структурированного лога (JSON) для парсинга пайплайнами.
- Критичные ошибки дублируются в click.echo(err=True), но не помечаются в логе как ERROR единообразно во всех командах.

## 6. Рекомендации

### 6.1 Документация

1. **Обновить QUICKSTART:** заменить устаревший блок «Пример использования» на актуальные примеры с подкомандами (`scan`, `upload-next`, `--delay` у команд загрузки).
2. **Обновить README:** добавить в список команд `folders`, `recalc-titles`, `update-vk-titles`; указать `scan -s mapped`, `--since`/`--until`.
3. **Обновить USAGE.md:** добавить разделы про маппинг папок (folders), источник `mapped`, фильтры по дате, пересчёт заголовков, update-vk-titles.
4. **Добавить docs/PIPELINES.md** (или раздел в USAGE): сценарии для OpenClaw (скан по датам, загрузка, обновление заголовков в VK), переменные окружения, коды выхода, где смотреть логи.

### 6.2 Пайплайны и коды выхода

1. В командах при фатальных ошибках вызывать `sys.exit(1)` (или click.exit(1)): отсутствие токена, неверный формат даты, отсутствие обязательного файла/записей для update-vk-titles.
2. Опционально: при частичном сбое (например, часть видео не загрузилась) — отдельный код (например, 2) по соглашению с OpenClaw.

### 6.3 Логирование

1. Поддержать переменную окружения `LOG_LEVEL` (например, DEBUG, INFO, WARNING) для переключения уровня без смены кода.
2. Опционально: флаг `--quiet` для команд, чтобы уменьшить вывод в stdout (оставив только итог), при этом в файл писать по-прежнему.

### 6.4 CLI

1. Добавить в справку по `scan` явное упоминание источника `mapped` (сканирование всех папок из маппинга БД).
2. Рассмотреть опцию `--db` для указания пути к `videos.db` при использовании в нескольких пайплайнах/окружениях.
