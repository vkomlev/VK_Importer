# Workflow: DB Change

## 1. Spec-Gate
- Описать бизнес-цель изменения БД.
- Зафиксировать инварианты данных.
- Определить требования к обратной совместимости.

## 2. Проектирование
- Подготовить миграцию (без ad-hoc DDL в прод-процессе).
- Оценить риски блокировок и объема данных.
- Подготовить план отката.

## 3. Execution-Gate
- Применить миграцию в тестовой среде.
- Проверить read/write сценарии, индексы, constraints.
- Зафиксировать результаты DB-check.

## 4. Review-Gate
- Архитектурное и безопасностное ревью миграции.
- Решение `PASS/FAIL`.

## 5. Merge-Gate
- При `PASS` выполнить merge в `main`.
- Приложить миграцию, результаты проверок и rollback-план.
